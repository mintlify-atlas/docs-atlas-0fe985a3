# solutionsData Array

The `solutionsData` array is a standalone export containing detailed information about VENCOL's product and service solutions. This data structure is separate from `siteContent.solutions.items` but shares the same `Service` type structure.

## Location

```
source/data/solutions.tsx
```

## Import

```tsx
import { solutionsData } from '@/data/solutions';
```

## Type Definition

```tsx
import { LucideIcon } from 'lucide-react';

type Service = {
  id: string;
  slug: string;
  title: string;
  description: string;
  longDescription: string;
  icon: LucideIcon;
  features: string[];
  image: string;
}

const solutionsData: Service[];
```

## Data Structure

### Service Object Properties

| Property | Type | Description |
|----------|------|-------------|
| `id` | `string` | Unique identifier (e.g., '1', '2', '3') |
| `slug` | `string` | URL-friendly identifier for routing |
| `title` | `string` | Display title of the solution |
| `description` | `string` | Short description (1-2 sentences) |
| `longDescription` | `string` | Detailed description (multiple paragraphs) |
| `icon` | `LucideIcon` | Icon component from lucide-react |
| `features` | `string[]` | Array of key features/benefits |
| `image` | `string` | Main image URL for the solution |

### 2. Solution Detail Page (Dynamic Route)

```tsx
import { solutionsData } from '@/data/solutions';
import { GetStaticPaths, GetStaticProps } from 'next';
import { Service } from '@/types';

type Props = {
  solution: Service;
};

export default function SolutionDetailPage({ solution }: Props) {
  const Icon = solution.icon;
  
  return (
    <div>
      <div className="flex items-center gap-4 mb-6">
        <Icon className="w-16 h-16 text-brand-green" />
        <h1 className="text-4xl font-bold">{solution.title}</h1>
      </div>
      
      <p className="text-xl text-gray-600 mb-8">{solution.description}</p>
      
      <div className="prose max-w-none mb-8">
        <p>{solution.longDescription}</p>
      </div>
      
      <img 
        src={solution.image} 
        alt={solution.title}
        className="w-full rounded-lg mb-8"
      />
      
      <h2 className="text-2xl font-bold mb-4">Características</h2>
      <ul className="space-y-2">
        {solution.features.map((feature, index) => (
          <li key={index} className="flex items-start">
            <span className="text-brand-green mr-2">✓</span>
            {feature}
          </li>
        ))}
      </ul>
    </div>
  );
}

export const getStaticPaths: GetStaticPaths = async () => {
  const paths = solutionsData.map((solution) => ({
    params: { slug: solution.slug },
  }));
  
  return { paths, fallback: false };
};

export const getStaticProps: GetStaticProps<Props> = async ({ params }) => {
  const solution = solutionsData.find((s) => s.slug === params?.slug);
  
  if (!solution) {
    return { notFound: true };
  }
  
  return {
    props: { solution },
  };
};
```

### 7. Search and Filter

```tsx
import { solutionsData } from '@/data/solutions';
import { useState } from 'react';

function SolutionsSearch() {
  const [searchTerm, setSearchTerm] = useState('');
  
  const filteredSolutions = solutionsData.filter((solution) =>
    solution.title.toLowerCase().includes(searchTerm.toLowerCase()) ||
    solution.description.toLowerCase().includes(searchTerm.toLowerCase())
  );
  
  return (
    <div>
      <input
        type="text"
        placeholder="Buscar soluciones..."
        value={searchTerm}
        onChange={(e) => setSearchTerm(e.target.value)}
        className="w-full p-3 border rounded mb-6"
      />
      
      <div className="grid gap-4">
        {filteredSolutions.map((solution) => (
          <div key={solution.id} className="border p-4 rounded">
            <h3 className="font-bold">{solution.title}</h3>
            <p className="text-sm text-gray-600">{solution.description}</p>
          </div>
        ))}
      </div>
    </div>
  );
}
```

## Relationship with siteContent

Note that `siteContent.solutions.items` (from `data/data.tsx`) contains the same solutions but with additional properties:

- `menuTitle` - Alternative title for menu display
- `subtitle1` and `subtitle1Description`
- `subtitle2` and `subtitle2Description`
- `images` - Array of multiple images instead of single `image`

Both arrays use the same `Service` type from `source/types`.

## Best Practices

### 1. Type Safety

```tsx
import { Service } from '@/types';
import { solutionsData } from '@/data/solutions';

type SolutionCardProps = {
  solution: Service;
};

function SolutionCard({ solution }: SolutionCardProps) {
  const Icon = solution.icon;
  return (
    <div>
      <Icon />
      <h3>{solution.title}</h3>
    </div>
  );
}
```

### 2. Icon Rendering

```tsx
// Always extract the icon component before rendering
const Icon = solution.icon;
<Icon className="w-6 h-6" />

// NOT like this:
{solution.icon} // This won't work
```

### 3. Static Path Generation

```tsx
// For Next.js static generation
export async function getStaticPaths() {
  return {
    paths: solutionsData.map((solution) => ({
      params: { slug: solution.slug }
    })),
    fallback: false
  };
}
```

### 4. SEO-Friendly URLs

```tsx
// Use slugs for clean URLs
// Good: /soluciones/bolsas-termoencogibles-cryovac
// Bad:  /soluciones/1

const url = `/soluciones/${solution.slug}`;
```

- Icons are functional React components, not strings or image URLs
- IDs are strings ('1', '2', etc.), not numbers
- Image URLs point to Unsplash placeholder images
- Features arrays contain 5 items each
